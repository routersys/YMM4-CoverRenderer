YukkuriMovieMaker4（YMM4）上で動作する、**音声・動画・画像ファイルのカバーアートをタイムライン上に図形として表示する図形プラグイン**です。
Windows の標準サムネイル API を利用してメディアファイルからカバー画像を自動取得し、Direct2D でレンダリングします。
ファイル選択には専用のカスタムセレクターを使用しており、同一ディレクトリ内のメディアファイルをサムネイルプレビュー付きのドロップダウンから素早く切り替えることができます。

## 概要

本プラグインは YMM4 の「図形」として動作します。指定したメディアファイル（音声・動画・画像）のカバーアートをサムネイルとして取得し、タイムライン上の図形アイテムとして描画します。

Windows Storage API によるサムネイル取得を音楽ビュー → 動画ビュー → 単一アイテムの順に試み、最初に取得できた画像を使用します。取得した画像は Direct2D のビットマップとしてレンダリングされ、`拡大率` パラメータによってキーフレームアニメーションが可能です。

選択したファイルと同一ディレクトリ内にあるメディアファイルが自動的にドロップダウンに列挙されるため、アルバムフォルダ内の曲を次々と切り替える操作も直感的に行えます。

## インストール方法

1. [Releases](https://github.com/yourname/CoverRenderer/releases/latest) ページから最新のプラグインファイル（`.ymme`）をダウンロードしてください。
2. YMM4 が起動していないことを確認し、ダウンロードしたファイルを実行（ダブルクリック）してインストールします。
3. YMM4 を起動し、タイムライン上に図形アイテムを追加します。
4. 図形の種類として **「カバーアート」** を選択してください。

## 主な機能

### 1. カバーアート表示

指定したメディアファイルのカバーアート（サムネイル）を最大 512px で取得し、タイムライン上の図形として描画します。

取得は Windows Storage API を使用し、以下の順序でサムネイルの取得を試みます。

- **音楽ビュー**: 音楽ファイルに埋め込まれたアルバムアートを取得します。
- **動画ビュー**: 動画ファイルのサムネイルフレームを取得します。
- **単一アイテム**: 上記で取得できなかった場合に汎用的なファイルサムネイルを取得します。

取得した画像は PBGRA32 形式に変換されたのち、Direct2D ビットマップとして GPU メモリ上に保持され、リニア補間でレンダリングされます。描画はキャンバス中央を原点とし、基準サイズ 512px（各辺 ±256px）を 100% 拡大率とします。

### 2. カスタムファイルセレクター

ファイルパスの選択には専用のカスタムセレクターを使用します。コンボボックスとフォルダ参照ボタンで構成されており、以下の機能を持ちます。

- **同一ディレクトリの自動列挙**: ファイルを選択すると、そのファイルと同一ディレクトリにある対応メディアファイルがコンボボックスに自動的に一覧表示されます。ファイルを開かずともドロップダウン操作だけで同フォルダ内のファイルを切り替えられます。
- **サムネイルプレビュー**: ドロップダウンを開くと各ファイルのサムネイルが 56px のプレビューとして表示されます。
- **「未選択」エントリ**: ドロップダウンの先頭には「未選択」エントリが常に表示されます。選択するとファイルパスがクリアされ、図形は空白になります。
- **参照ボタン**: フォルダアイコンのボタンをクリックすると「ファイルを開く」ダイアログが表示されます。一覧に含まれていない任意のファイルを直接指定できます。

### 3. 拡大率のアニメーション

`拡大率` パラメータはキーフレームアニメーションに対応しています。時間経過に合わせてカバーアートの表示サイズを滑らかに変化させることができます。

## 対応ファイル形式

カスタムファイルセレクターのドロップダウンに自動列挙される対応拡張子は以下のとおりです。

| カテゴリ | 拡張子 |
|---|---|
| 動画 | `.mp4` `.avi` `.mkv` `.mov` `.wmv` `.webm` `.flv` |
| 音声 | `.mp3` `.wav` `.m4a` `.flac` `.ogg` `.aac` `.wma` `.opus` |
| 画像 | `.png` `.jpg` `.jpeg` `.bmp` `.gif` `.tif` `.tiff` `.webp` |

参照ボタンから開く「ファイルを開く」ダイアログでは、上記以外のファイルも選択可能です。ただし、Windows がサムネイルを生成できない形式の場合はカバーアートが表示されません。

## パラメータ一覧

| パラメータ名 | 型 | デフォルト | 範囲 | アニメーション | 説明 |
|---|---|---|---|---|---|
| **ファイルパス** | ファイル選択 | (未選択) | — | ✗ | 表示するカバーアートの元となるメディアファイルを指定します |
| **拡大率** | 数値 (%) | 100 | 0〜5000 | ✔ | カバーアートの表示サイズを基準サイズ（512px）に対するパーセントで指定します |

## 使い方ガイド

### 基本的な使い方

1. タイムライン上に図形アイテムを追加します。
2. 図形の種類として **「カバーアート」** を選択します。
3. **ファイルパス** のセレクターで、表示したいメディアファイルを選択します。
   - フォルダアイコンのボタンをクリックするとファイル選択ダイアログが開きます。
   - 一度ファイルを選択すると、同一フォルダ内のメディアファイルがドロップダウンに自動的に列挙されます。
4. **拡大率** を調整してカバーアートのサイズを変更します。

### アルバムアートとして使う

楽曲ファイルのアルバムアートを表示したい場合の手順です。

1. MP3・M4A・FLAC などの音楽ファイルをあらかじめ用意し、ID3 タグなどにアルバムアートを埋め込んでおきます。
2. ファイルパスセレクターでその音楽ファイルを選択します。
3. Windows がファイルからアルバムアートを認識できれば、自動的に取得・表示されます。

### 拡大率をアニメーションさせる

カバーアートを徐々に拡大・縮小する演出に使用できます。

1. `拡大率` パラメータのキーフレームを複数設定します（例：0フレーム目に 0%、30フレーム目に 100%）。
2. フレーム間で自動的に補間され、滑らかな拡大・縮小アニメーションが得られます。

### 複数のファイルを素早く切り替える

同一フォルダ内の複数ファイルを切り替えたい場合、ドロップダウンを使うと効率的です。

1. 最初のファイルを参照ボタンから選択します。
2. 以降はドロップダウンを開くと同フォルダ内のメディアファイルが一覧表示されるため、ダイアログを使わずに切り替えられます。

## 注意事項

- **AviUtl 非対応**: 本プラグインは AviUtl 形式（`.exo`）への出力に対応していません。マスクとしての出力にも対応していません。
- **サムネイル非対応ファイル**: Windows がサムネイルを生成できないファイルを指定した場合、カバーアートは表示されず図形が空白になります。
- **ファイルパス変更時の再取得**: ファイルパスを変更するたびにサムネイルの再取得処理が実行されます。大量のキーフレームでファイルパスを頻繁に切り替えると処理に時間がかかる場合があります。
- **OS バージョン**: Windows Storage API を使用しているため、Windows 10 バージョン 2004（ビルド 19041）以降が必要です。それ以前の OS バージョンでは動作しません。
- **本プラグインを使用する前に、YMM4 プロジェクトファイルのバックアップを作成することを推奨します。**
